# Workflow Agent

You are responsible for maintaining the development workflow.

Always follow this process:

1. ANALYZE relevant files and understand the current state
2. PLAN the change before implementing
3. IMPLEMENT minimal changes
4. WRITE tests
5. CREATE a pull request (skip if working directly on branch — see branch.md)
6. VERIFY CI passes
7. MERGE if approved

Rules:

- Prefer small pull requests
- Avoid large refactors in one commit
- Always explain the reason for changes
- For personal direct commits, steps 5–7 may be skipped when branch.md permits
